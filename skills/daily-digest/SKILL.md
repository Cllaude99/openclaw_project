# 일일 종합 브리핑 스킬

## 사용 시점

- 크론 작업에 의한 자동 트리거 (아침 07:00, 점심 12:30, 저녁 18:00)
- 사용자가 "브리핑", "요약", "종합" 등을 요청할 때

## 브리핑 유형

### 1. 아침 브리핑 (07:00 KST)
코인 시세 + AI 코인 추천 + 주식 시세 + AI 종목 추천 + 뉴스 종합

### 2. 점심 뉴스 (12:30 KST)
IT 뉴스 + 테크블로그 최신 글 + 국내 주요 뉴스 중심

### 3. 저녁 시장 요약 (18:00 KST, 평일)
코인 시세 + 국내/해외 주식 시세 + 하루 변동 분석

### 4. 주간 채용 (월요일 09:00 KST)
Frontend 채용 정보

## 실행 절차

다음 순서로 데이터를 수집합니다:

### Step 1: 암호화폐 데이터 수집
```bash
bash skills/crypto-kr/scripts/fetch_crypto.sh
bash skills/crypto-kr/scripts/fetch_crypto_analysis.sh
```

### Step 2: 주식 데이터 수집
```bash
bash skills/stocks-kr/scripts/fetch_stocks.sh
```

### Step 3: 기술 뉴스 수집
```bash
bash skills/tech-news/scripts/fetch_hn.sh
```

### Step 4: 테크블로그 수집
```bash
bash skills/tech-blog-kr/scripts/fetch_tech_blogs.sh
```

### Step 5: 일반 뉴스 수집
```bash
bash skills/news-summary/scripts/fetch_news.sh
```

### Step 6: (월요일만) 채용 정보 수집
```bash
bash skills/jobs-kr/scripts/fetch_jobs.sh
```

### 점심 뉴스 (IT + 테크블로그 + 국내)
```
📅 1월 15일 점심 뉴스

[각 기사에 🔗 원문 링크 + 💬 HN 토론 링크 포함]

📝 테크블로그 최신 글
[각 글에 → 원문 링크 포함]

📰 국내 주요 뉴스
[각 뉴스에 🔗 기사 링크 포함]
```

**중요**: 뉴스, 기술 글, 테크블로그, 채용 등 원문이 있는 모든 정보에는 반드시 클릭 가능한 링크를 포함한다. 브리핑에서도 링크를 생략하지 않는다.

## 메시지 제한 처리

- Telegram 메시지 제한: 4096자
- `textChunkLimit: 4000`으로 설정되어 자동 분할
- 분할 시 각 메시지가 의미 있는 단위로 나뉘도록 섹션 단위로 구성
- 아침 브리핑: 1메시지(코인+주식) + 2메시지(뉴스+테크블로그)로 분할

## 오류 처리

- 특정 API 실패 시 해당 섹션만 "현재 데이터를 가져올 수 없습니다"로 표시
- 나머지 섹션은 정상 출력
- 전체 API 실패 시 "브리핑 생성에 실패했습니다. 잠시 후 다시 시도해주세요."
